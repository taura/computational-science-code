# 追加問題の提案 (タイプ2: 多少なりとも意味のある計算をする問題)

各トピックに, 「実際に意味のある数値計算」を題材にした問題を提案する。
タイプ1 (工夫問題) と同様, そのトピックまでの知識だけで解けることを前提とし, 既存問題 (00, 01) と重複しないものを選ぶ。
無理に作らない (題材が乏しい所は「限定的/見送り」と明記)。番号は既存の続き (多くは `02_`) を想定。

motif (使い回す題材): 行列ベクトル積 / 行列積 / マンデルブロ集合 / 数値積分 / モンテカルロ / 内積。
これらは「並列化・GPU化・SIMD化の効果が見える」典型例。

---

## 02_parallel — (限定的) `02_montecarlo_partial` モンテカルロ π (各スレッドが部分推定)

- **意味のある計算**: 単位正方形に点を撒き, 単位円内に落ちた割合から π を推定する (モンテカルロ法)。
- **やること**: `for` も `reduction` もまだ無いので, 各スレッドが `omp_get_thread_num()` を乱数の種にして自分の分の点を撒き, **自分の π 推定値を表示する** (集約はしない)。
- **使う知識**: `parallel` + スレッド番号のみ。
- **位置づけ**: タイプ1 `01_manual_partition` と発想が同じ (手分け+各自表示) なので, **重複気味。02 は見送りでもよい**。作るならモンテカルロという題材で意味を持たせる程度。

## 03_for_collapse — `02_matvec` 行列ベクトル積 y = A x

- **意味のある計算**: 密行列 A (n×n) とベクトル x の積 y = A x。数値計算の最重要カーネルの一つ。
- **やること**: 外側 (行 i) のループを `#pragma omp parallel for` で並列化。各行の内積 `y[i] = Σ_j A[i][j]*x[j]` は内側ループ (各 i で独立, 共有変数なし) で計算するので reduction 不要。
- **発展 (collapse)**: 行列の初期化 `A[i][j] = f(i,j)` の二重ループを `collapse(2)` で並列化する小問を添える。
- **使う知識**: parallel for, collapse, 2次元配列。
- **意味**: 結果を検算 (例: A を単位行列にすると y=x) でき, 「意味のある計算が並列化できた」実感。

## 04_schedule — `02_mandelbrot` マンデルブロ集合 (負荷不均一の代表例)

- **意味のある計算**: 複素平面の格子点ごとに `z ← z² + c` を反復し, 発散するまでの回数 (escape count) を求める。集合の内側は最大反復までまわり, 外側はすぐ抜ける → **点ごとに仕事量が大きく異なる**。
- **やること**: 各ピクセルを独立に計算 (結果を配列に格納)。`schedule(static)` だと画像の「重い帯」を担当したスレッドに偏る → `schedule(dynamic)` で改善。実行時間を比較。
- **使う知識**: parallel for, schedule, 複素数を実部・虚部で手書き, 2次元配列。
- **意味**: 負荷不均衡が「絵」として直感的に分かる, schedule の威力が一番見える題材。`01_count_primes` と狙いは近いが, より視覚的で本格的。

## 05_speedup — `02_matmul` 行列積 C = A B の台数効果

- **意味のある計算**: 行列積 (n×n)。HPC の代表的ベンチマーク。FLOPS = 2 n³。
- **やること**: 三重ループの一番外側を `#pragma omp parallel for` で並列化し, n を変えながらスレッド数ごとの GFLOPS を測る。`OMP_PROC_BIND=true`。
- **使う知識**: parallel for, 台数効果の測定, 2次元配列。
- **意味**: 「教科書のキラーアプリ」で台数効果を測れる。`00_measure_speedup` (人工的な lin_rec) より実用的な題材で, スケーリングとキャッシュの効き方も観察できる。

## 06_reduction — `02_montecarlo_pi` モンテカルロ π (count を reduction)

- **意味のある計算**: 単位円内に落ちた点の数を数え π を推定。`reduction(+:count)`。
- **やること**: `n` 個の乱数点について円内判定し, `count` を reduction で集計 → π ≈ 4·count/n。誤差が n とともに減ることを観察。
- **使う知識**: parallel for, reduction。
- **意味**: `00_reduction_sum` (数値積分) とは別アルゴリズムで π を求める。乱数の扱い (スレッドごとに種を変える/`rand_r` 等) という実践的な注意も学べる。
- 代替案: **内積 / ベクトルノルム** `s = Σ x[i]*y[i]` も良い (より単純, 数値計算頻出)。どちらか選択。

## 07_gpu_target — (見送り)

- `target` 単体 (map なし) で意味のある計算は難しい。`08`/`09` を待つ。**見送り**。

## 08_gpu_teams — `02_gpu_integral` GPUで数値積分 (scalar reduction)

- **意味のある計算**: `∫₀¹ 4/(1+x²) dx = π` を矩形則で。
- **やること**: `#pragma omp target teams distribute parallel for reduction(+:s)` で総和を GPU 上で計算。**結果はスカラ `s` なので明示的な `map` は不要** (スカラは暗黙にやり取りされる) → map (09) を学ぶ前でも成立。
- **使う知識**: target teams distribute parallel for, reduction。
- **意味**: GPU で「答えのある計算」(π) を出せる。チーム数・スレッド数で速さが変わることも観察。

## 09_gpu_map — `02_gpu_matvec` GPUで行列ベクトル積 (map で配列を授受)

- **意味のある計算**: y = A x を GPU で。
- **やること**: `map(to: A[0:n*n], x[0:n])` で入力を送り, `map(from: y[0:n])` で結果を受け取る。検算 (A=単位行列など)。
- **使う知識**: target teams distribute parallel for, map(to/from), 2次元配列の1次元化。
- **意味**: `01_vecadd` の次として, 入力が「行列+ベクトル」の現実的なカーネル。転送するデータの大きさ (n² と n) の違いも意識できる。

## 10_gpu_speedup — `02_gpu_matmul` GPUで行列積の性能測定 (CPU比較)

- **意味のある計算**: 行列積 C = A B を GPU で。
- **やること**: GPU 版の GFLOPS を測り, `05_matmul` の CPU 最高性能と比較。問題サイズ n を変えて GPU が有利になる領域を見る。
- **使う知識**: target teams distribute parallel for, map, 性能測定。
- **意味**: 05 (CPU) と 10 (GPU) で同じ行列積を比較でき, トピックがつながる。`01_cpu_vs_gpu` の本格版。

## 11_simd — `02_dot_simd` 内積を omp simd で (simd reduction)

- **意味のある計算**: 内積 `s = Σ x[i]*y[i]`。
- **やること**: `#pragma omp simd reduction(+:s)` で総和ループをSIMD化。`-Minfo`/`.s` で packed 命令と水平加算を確認。
- **使う知識**: omp simd, reduction (simd reduction)。
- **意味**: 単なる要素ごと演算 (00 の saxpy) と違い, **縮約を伴うSIMD化** という一歩進んだ題材。Fortran も `!$omp simd reduction` または配列組込み関数 `dot_product` で対比。

## 12_simd_intrinsics — `02_poly_horner` 多項式評価をベクトル型で

- **意味のある計算**: 配列の各 `x[i]` について多項式 `p(x) = c0 + c1 x + c2 x² + …` を Horner 法で評価。
- **やること**: ベクトル型で `nl` 個ずつ, Horner の漸化 `acc = acc*x + c_k` をベクトルのまま回す。
- **使う知識**: ベクトル型, load/store, fma。
- **意味**: `01_map_kernel` (1段の式) の発展で, fma が効く実用的カーネル。関数近似などで頻出。Fortran は `!$omp simd` + Horner ループで対比。

## 13_ilp — `02_matmul_peak` 行列積で1コア/全コアの限界性能に挑む

- **意味のある計算**: 行列積の最内カーネルを, ベクトル型 (SIMD) + `nl` 調整 (ILP) + `#pragma omp parallel for` (マルチコア) で最適化し, ピーク GFLOPS を目指す。
- **やること**: ブロック化した行列積カーネルをベクトル型で書き, nl・スレッド数を調整。理論ピーク (コア数 × SIMD幅 × FMA × クロック) に対する達成率を評価。GPU (10) とも比較。
- **使う知識**: ベクトル型, ILP, マルチコア, 性能測定 (総動員)。
- **意味**: コース全体の集大成。「意味のある計算 (行列積) で, CPU の限界性能とは何かを体感する」。
- **注意**: やや重い。`00`/`01` の lin_rec ベースで十分なら, 行列積版は発展課題として任意でもよい。

# 応用・発展問題の仕様案 (意味のある計算)

π などの定番を越えた「計算して嬉しい/面白い」題材の **問題仕様のみ** をまとめる (プログラムはまだ書かない)。
各題材について: 概要 / 計算の中身 / 入力・パラメータ / 出力・検算 / 並列化のポイント (使う技法) / 難易度 / ベースライン案 (逐次コードを与えて何を並列化・穴埋めさせるか)。

難易度の目安: ★ 易 〜 ★★★★★ 重。本コースは情報系でない学生向けなので, 重いものは「逐次コードを与えて並列化だけさせる」「一部だけ切り出す」等のスキャフォルディングを想定。

凡例: 技法 = parallel / for / collapse / schedule / reduction / GPU(target,teams,map) / SIMD(omp simd, ベクトル型) / ILP。

# A. モンテカルロ (日常の確率・期待値)

## A1. ガチャ (クーポンコレクター)  ★
- **概要**: N 種類の景品が等確率で出るガチャを, 全種類そろうまで引く。必要回数の期待値・分布を推定。
- **計算の中身**: 1試行 = 「全種そろうまで引いた回数」を数える。多数試行の平均・分散・ヒストグラム。理論期待値は N·H_N (H_N = 調和数)。
- **入力**: 種類数 N, 試行回数 T。
- **出力・検算**: 平均引き回数 ≈ N·H_N (例 N=10 → 約29.3)。分布も表示。
- **並列化**: 試行 T 個は独立 → parallel for + reduction(平均用の和)。乱数はスレッドごとに種を変える。
- **難易度**: ★ (reduction の意味のある応用)
- **ベースライン案**: 逐次で1試行を行う関数を与え, 外側の試行ループを並列化＋reduction を埋めさせる。

## A2. 待ち合わせ問題  ★
- **概要**: 2人が 0〜60 分の一様乱数時刻に来て, 先に来た方は t 分だけ待つ。会える確率。
- **計算の中身**: 試行ごとに |x−y| ≤ t を判定し, 該当数 / 試行数。理論値あり (t=15 で約 7/16)。
- **入力**: 待ち時間 t, 試行回数。
- **出力・検算**: 会える確率 ≈ 1−((60−t)/60)²。
- **並列化**: parallel for + reduction(+:count)。
- **難易度**: ★

## A3. 金融: ヨーロピアン・コールの価格  ★★
- **概要**: 株価を幾何ブラウン運動でシミュレートし, 満期ペイオフ max(S_T−K,0) の期待値 (割引後) でオプション価格を推定。
- **計算の中身**: 各パスで S_T = S_0·exp((r−σ²/2)T + σ√T·Z), Z は標準正規乱数。多数パスのペイオフ平均 × e^{−rT}。
- **入力**: S_0, K, r, σ, T, パス数。
- **出力・検算**: Black–Scholes 解析解と一致 (誤差はパス数で減る)。
- **並列化**: パスは独立 → parallel for + reduction。正規乱数の生成 (Box–Muller)。
- **難易度**: ★★ (金融の実用例, 解析解と比較できる)

## A4. パーコレーション (浸透)  ★★★
- **概要**: L×L 格子の各セルを確率 p で「通れる」にする。上端から下端まで通れる (浸透する) 確率を p ごとに推定。
- **計算の中身**: ランダム格子を作り, 連結判定 (BFS/DFS か Union-Find) で上下が繋がるか。多数試行で浸透確率。**p をスイープすると臨界点 p_c≈0.5927 付近で確率が急変 (相転移) する**。
- **入力**: 格子サイズ L, 確率 p (複数), 試行回数。
- **出力・検算**: 浸透確率 vs p の曲線 (S字)。臨界点付近の立ち上がり。
- **並列化**: 試行が独立 → parallel for (各試行内の連結判定は逐次)。schedule(dynamic) (試行ごとに手間が違う)。
- **難易度**: ★★★ (連結判定の実装が要る。相転移が見えて嬉しい)
- **ベースライン案**: 1試行 (格子生成+連結判定) の逐次関数を与え, 試行ループの並列化と schedule を考えさせる。

# B. 偏微分方程式・差分法 (stencil)

## B1. 2D 定常熱伝導 (ラプラス方程式, ヤコビ反復)  ★★
- **概要**: 四辺の温度を固定した正方形の板。内部温度が定常分布に落ち着くまで反復。
- **計算の中身**: 各格子点を上下左右4点の平均で更新 u_new[i][j] = (u[i±1][j]+u[i][j±1])/4 を, 変化が十分小さくなるまで反復 (ヤコビ法)。
- **入力**: 格子サイズ N, 境界温度, 収束判定しきい値 (または反復回数)。
- **出力・検算**: 定常温度分布 (中心の値など)。簡単な境界条件なら解析解/対称性で検算。
- **並列化**: 内側の二重ループを collapse(2) で並列化。収束判定の最大残差は reduction(max)。GPU(target teams distribute) でも。台数効果も測れる。
- **難易度**: ★★ (差分法の王道。collapse/schedule/GPU/台数効果を一本で網羅できる主役候補)
- **ベースライン案**: 逐次のヤコビ反復を与え, 更新ループの collapse 並列化＋残差 reduction を埋めさせる。GPU版/台数効果版に派生可。

## B2. 2D 波動方程式 (陽解法)  ★★
- **概要**: 膜/水面の波が時間とともに広がる様子。
- **計算の中身**: u^{n+1} = 2u^n − u^{n−1} + c²Δt²·(ラプラシアン)。時間ステップを進める。
- **入力**: 格子サイズ, ステップ数, 初期波形, 波速 c。
- **出力・検算**: 波が反射・干渉する様子 (スナップショット出力)。エネルギー保存の概算。
- **並列化**: 空間の二重ループを collapse(2)。時間方向は逐次 (依存)。
- **難易度**: ★★

## B3. インクの拡散 (2D 拡散方程式)  ★★
- **概要**: 水中のインクが時間とともに広がる。
- **計算の中身**: u^{n+1}[i][j] = u^n + D·Δt·(ラプラシアン)。
- **並列化**: B1/B2 と同様の stencil。
- **難易度**: ★★

# C. N体問題

## C1. 重力 N体 (直接法 O(N²))  ★★
- **概要**: N 個の質点が万有引力で相互作用しながら動く (太陽系/星団)。
- **計算の中身**: 各粒子 i について他の全粒子 j からの力 Σ_j G·m_i·m_j·(r_j−r_i)/|r_j−r_i|³ を計算 (二重ループ, ソフトニング付き) → 速度・位置を時間積分 (leapfrog/Euler)。
- **入力**: 粒子数 N, ステップ数, 初期配置。
- **出力・検算**: 軌道のスナップショット。全エネルギー/運動量の保存度。
- **並列化**: 力計算の外側ループ (粒子 i) を parallel for。内側の和は局所変数に集約 (reduction 不要)。SIMD・GPU でさらに加速。O(N²) なので台数効果が綺麗。
- **難易度**: ★★ (HPC の定番。台数効果・SIMD の見せ場)
- **ベースライン案**: 逐次の力計算＋積分を与え, 力計算ループの並列化を埋めさせる。SIMD/GPU/台数効果に派生。

## C2. Boids (群れの創発)  ★★
- **概要**: 各個体が「分離・整列・結合」の近傍ルールで動くと, 鳥や魚の群れが創発する。
- **計算の中身**: 各個体について近傍個体を見て3つの力を合成 → 速度更新。素朴には O(N²)。
- **出力**: 群れの動きのスナップショット (見て楽しい)。
- **並列化**: 個体ループを parallel for。
- **難易度**: ★★ (ビジュアル重視, 日常的で楽しい)

## C3. バネ質点系 (布・ロープ)  ★★
- **概要**: 質点をバネで繋いだ布が重力で垂れる/揺れる。
- **計算の中身**: 各バネの力 → 各質点の加速度 → 時間積分。
- **並列化**: 質点/バネのループを並列化 (近傍依存に注意)。
- **難易度**: ★★

# D. AI (MLP / MNIST 系)  ※ LLM は別途 qwen from scratch のため対象外

## D1. MNIST 推論 (学習済み重みで forward のみ)  ★★
- **概要**: 学習済みの重みを読み込み, 手書き数字画像 (28×28) を多層パーセプトロンに通して数字を判定。
- **計算の中身**: 各層 = 行列ベクトル積 (または バッチで行列積) + バイアス + 活性化 (ReLU/softmax)。**中身はこれまで作った行列(ベクトル)積そのもの**。
- **入力**: 重みファイル, テスト画像 (バッチ)。
- **出力・検算**: 認識結果と正解率 (テストセットで ~97% 等)。
- **並列化**: 行列積を parallel for / collapse / GPU / SIMD。バッチ方向の並列も。
- **難易度**: ★★ (「行列積 = AI」に着地。03/05/10/13 の集大成的応用)
- **ベースライン案**: forward の逐次実装 (matmul + 活性化) を与え, matmul を並列化させる。データ・重みは用意して配布。

## D2. k-means クラスタリング  ★★
- **概要**: 点群を K 個のクラスタに自動分類。
- **計算の中身**: (1) 各点を最近傍の重心に割当 (距離計算), (2) 重心を再計算, を収束まで反復。
- **入力**: 点群, K。
- **出力・検算**: クラスタ割当, 重心。
- **並列化**: 割当ステップ (点ごと) を parallel for。重心の和は reduction。
- **難易度**: ★★

## D3. ロジスティック回帰 / 線形回帰の学習  ★★
- **概要**: データから勾配降下でモデルを学習し, 学習曲線 (損失の低下) を見る。
- **計算の中身**: 予測 (matvec) → 勾配 (reduction) → パラメータ更新, を反復。
- **並列化**: matvec とデータ和の reduction。
- **難易度**: ★★ (学習が進む達成感, MLP への足がかり)

## D4. MLP 学習 (SGD, ミニバッチ)  ★★★
- **概要**: 2〜3層のニューラルネットを MNIST で自分で学習させ, 精度が上がる様子を見る。
- **計算の中身**: forward (matmul) → 損失 → backprop (matmul) → 更新。
- **並列化**: 行列積中心。バッチ並列。
- **難易度**: ★★★ (集大成級。backprop の実装が要る)

# E. 偏微分方程式・有限要素法 (FEM)

## E1. 1D/2D ポアソン方程式の FEM  ★★★★ 【保留: 重いので G1 CG法単体 で代替】
- **概要**: 有限要素法で −∇²u = f を解く (定常熱/弾性の簡単版)。
- **計算の中身**: (1) メッシュ分割, (2) 各要素の剛性行列を計算して全体行列に組み立て (assembly), (3) 線形方程式を反復解法 (共役勾配法 CG) で解く。
- **並列化**: assembly は要素ループの並列化 (書き込み競合に注意), CG は疎行列ベクトル積 (SpMV) + reduction(内積) の並列化。
- **難易度**: ★★★★ (メッシュ・疎行列・CG とフルセットで重い)
- **スキャフォルディング案**: フルFEMは重いので, **(a) assembly だけ**, **(b) CG の SpMV だけ** を切り出した問題にするのが現実的。完全版は発展課題。

# G. 線形代数カーネル (反復解法・固有値)

差分法/FEM や AI の土台になる「行列に対する反復計算」。中身は既習の **行列ベクトル積・内積(reduction)・axpy** の組合せなので, FEM 完全版より軽く, 並列化の対象がはっきりしている。

## G1. 共役勾配法 (CG) で連立一次方程式を解く  ★★★ 【採用 / E1(FEM)の代替】
- **概要**: 対称正定値 (SPD) 行列 A に対し A x = b を反復解法 CG で解く。差分法・FEM で現れる大規模疎行列の標準解法。
- **計算の中身**: 反復ごとに「行列ベクトル積 q = A p」「内積 (p·q, r·r)」「axpy (x += α p, r −= α q)」を繰り返し, 残差が小さくなれば収束。
- **入力**: 行列 A (例: **2D ラプラシアンの5点ステンシル → B1 と接続**。メッシュ不要で生成でき軽い), 右辺 b, 収束しきい値。
- **出力・検算**: 解 x。残差 ||A x − b|| → 0。既知解 (例: b を既知 x から作る) と一致。
- **並列化**: 行列ベクトル積 (またはSpMV) を parallel for, 内積を reduction, axpy を parallel for。GPU/SIMD にも展開可。
- **難易度**: ★★★ (反復解法の全体構造を理解する。各カーネルは既習部品の組合せ)
- **ベースライン案**: 逐次 CG を与え, **matvec と内積(reduction)** を並列化させる。行列は 2D ラプラシアンを内部生成 (FEMメッシュ不要)。発展で SpMV/GPU 版。

## G2. べき乗法で固有値・固有ベクトルを求める  ★★ 【採用】
- **概要**: 行列 A をベクトルに繰り返し掛けて正規化すると, (絶対値) 最大固有値の固有ベクトルに収束する (べき乗法)。中身は **matvec + 内積/ノルム (reduction)** の反復で, G1 CG法と同じ構造。
- **計算の中身**: x ← A x → レイリー商 λ = (x·A x)/(x·x) → x ← x/||x|| を反復。
- **並列化**: 行列ベクトル積 (parallel for) + 内積/ノルム (reduction)。
- **難易度**: ★★ (matvec + reduction の自然な応用)
- **着地点を2つ用意 (両方採用)** — 「結果を味わえる」題材:

  ### G2-① 弦・膜の固有振動モード  (イチオシ)
  - **概要**: ラプラシアン行列の固有ベクトル = 定在波の形 (弦の倍音, 太鼓の振動模様)。**B1/G1 と同じ行列**を使える。
  - **計算の中身**: 1D なら3点ステンシル (−1,2,−1) のラプラシアン L。基本モード (なめらかな半波) を出すには `(σI − L)` にべき乗法を当てる (σ は十分大きい定数 → matvec に引き算が増えるだけ)。最大固有のまま (最高周波のギザギザ) でも形は出る。
  - **入力**: 格子点数 N, シフト σ, 反復回数。
  - **出力・検算**: モードの「形」 (配列を出力 → プロット)。1D 基本モードは理論 sin(πx) と一致。2D にすると太鼓の模様。
  - **接続**: 「**同じラプラシアンを, B1 で解く / G1 で満たす / G2 で振動させる**」が一本につながる。

  ### G2-② すごろく/ランダムウォークの定常分布
  - **概要**: マスの遷移ルールを自前で決め, 遷移行列の最大固有ベクトル (=定常分布) をべき乗法で求める。「**長く遊ぶと一番よく止まるマス**」のランキング。PageRank の遊べる版 (実データ不要)。
  - **計算の中身**: 確率ベクトル p ← (遷移行列) p を収束まで反復 (最大固有値=1)。シフト不要の純粋なべき乗法。
  - **入力**: マス数とサイコロ/ワープ等のルール (自前生成), 反復回数。
  - **出力・検算**: 各マスの定常確率 (合計1) と順位。小さい盤で手計算/直感と照合。
  - **接続**: 06 のモンテカルロ/確率の系譜。同じ「止まりやすさ」を A 系のシミュレーションでも近似でき, 突き合わせると面白い。

## G3. PageRank  ★★ 【保留: G2-② すごろく定常分布で代替】
- ウェブのリンク構造に対するべき乗法。実体は G2-② と同じ (確率行列の定常分布)。実データが要る/やや飽きるため, 遊べる **G2-② すごろく**に置き換えて保留。記録として残す。

## G4. (発展) 対称行列の全固有値 — ヤコビ法 / ランチョス法  ★★★★ 【保留】
- **概要**: 1つではなく全固有値を求める古典解法。回転 (ヤコビ) や部分空間 (ランチョス) を使う。
- **難易度**: ★★★★ (アルゴリズムが重い。発展課題として記録のみ)
